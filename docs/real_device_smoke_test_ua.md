# Smoke-test Milestones на реальному телефоні

Мета: перевірити те, чого не гарантує desktop/headless viewport — браузерні панелі, safe area,
standalone PWA, відновлення після закриття та локальне сховище. Це технічна перевірка, не
батьківське UX-дослідження і не експертна оцінка контенту.

Поточна база: **Stage 5 UA / P2.45**, `20260704-p2-45-r1`. Перед тестом звірити
[`../CURRENT_BUILD.md`](../CURRENT_BUILD.md).

## Пристрої

- [ ] Android + актуальний Chrome.
- [ ] iPhone + актуальний Safari.

Для кожного пристрою записати модель, версію ОС, браузер і дату.

## Короткий сценарій

1. Відкрити актуальну Stage 5 URL через HTTPS або локальну приватну адресу, доступну телефону.
2. Додати застосунок на домашній екран і запустити його звідти.
3. Створити лише тестовий профіль без реальних персональних даних.
4. На головній, у Спостереженні, Грі, Бібліотеці та Motion Cards перевірити:
   - [ ] немає глобального горизонтального прокручування або обрізання справа;
   - [ ] нижня навігація не перекривається системною панеллю/safe area;
   - [ ] основні кнопки легко натискаються одним великим пальцем;
   - [ ] поля не спричиняють небажаний zoom і залишають активну дію видимою.
5. Завершити одну тестову відповідь/дію, повністю закрити застосунок і відкрити знову:
   - [ ] тестовий профіль і прогрес залишилися локально;
   - [ ] застосунок повертається у зрозумілий стан без порожнього екрана.
6. Після одного успішного онлайн-запуску вимкнути мережу й відкрити PWA:
   - [ ] оболонка, основні маршрути та Motion Cards відкриваються офлайн;
   - [ ] видно спокійний статус офлайн, а локальні дії не губляться.
7. У згорнутих налаштуваннях даних:
   - [ ] створити локальний backup тестового профілю;
   - [ ] відновити його лише після явного підтвердження;
   - [ ] переконатися, що файл не містить реальних даних дитини.
8. Коли доступне оновлення service worker:
   - [ ] застосунок не перезавантажується самовільно;
   - [ ] оновлення застосовується лише після «Оновити зараз».

## Результат

Для кожної проблеми записати: пристрій, маршрут, точні кроки, очікуваний і фактичний результат,
скріншот та критичність (`blocker / major / minor`). Якщо обидва пристрої проходять сценарій,
позначити physical-device smoke gate як пройдений у `HANDOFF.md`.
