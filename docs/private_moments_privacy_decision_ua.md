# E6 — privacy-рішення для «Маленьких моментів»

Дата: 2026-07-03  
Версія: P2.30  
Рішення: локальна текстова добірка без фото, хмари, акаунта й аналітики

## Що реалізовано

- перевикористовуються вже наявні необов’язкові нотатки після гри;
- записи ізольовані за профілем дитини в локальному сховищі браузера;
- на головній показуються лише три останні записи, без нескінченного скролу;
- немає лайків, серії, балів, порівнянь або публічного поширення;
- кожен запис можна видалити після явного підтвердження;
- записи входять до приватної резервної копії, про що інтерфейс повідомляє прямо;
- календар створює лише одну подію на наступний день, без прихованого повторення.

## Мінімізація даних

Нової таблиці, ідентифікатора, фото, геолокації чи часової аналітики не створюється. Використовується ключ на кшталт `YYYY-MM-DD:age`, назва виконаної гри й текст нотатки до 1000 символів. У UI видно максимум три записи; решта потрібні лише для вже наявної історії та підсумку фахівцю.

## Ризики й межі

| Ризик | Поточне зменшення | Що забороняє хмарний запуск |
|---|---|---|
| Інша людина має доступ до пристрою | локальне зберігання, індивідуальне видалення | потрібні модель загроз, автентифікація та шифрування |
| У вільній нотатці випадково записано персональні дані | немає автоматичного поширення; backup попереджає про приватність | перед будь-яким share/export потрібен preview і явна згода |
| Втрата браузерних даних | ручна резервна копія | синхронізація не додається без окремої згоди й політики зберігання |
| Функція перетворюється на оцінювання | нейтральний текст, лише три записи, без метрик прогресу | заборонені streak, рейтинг, «норма моментів» і нагороди |
| Нагадування створює тиск | одна локальна календарна подія | повторні push-нагадування потребують окремого opt-in і тесту тону |

## Дані й життєвий цикл

- створення: лише коли дорослий сам вводить нотатку після позначеної гри;
- читання: головна, історія/підсумок і локальна резервна копія;
- виправлення: редагування у поточному полі нотатки в день гри;
- видалення: окремий запис із головної або весь профіль через керування даними;
- передавання: відсутнє;
- retention: автоматичне стирання не застосовується, доки не перевірено очікування батьків; показ обмежено трьома останніми.

## Gate перед наступним розширенням

1. П’ять батьків розуміють, що записи локальні й не є оцінкою.
2. Видалення та резервна копія зрозумілі без додаткового пояснення.
3. Емоційний тест не знаходить провини або тиску «записувати щодня».
4. Фото, синхронізація, share, push і довший feed залишаються забороненими до окремого privacy review.
